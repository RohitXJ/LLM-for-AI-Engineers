SRE-Pulse: Project System Requirements Specification (SRS)
=========================================================

Project Structure
-----------------
Project/
├── main.py              # Entry point: Streamlit Dashboard / CLI Loop
├── data/
│   └── sre_docs.json     # Knowledge base: Technical SRE documents
├── core/
│   ├── __init__.py
│   ├── engines.py        # Logic for Keyword (BM25) and Vector (Chroma) search
│   ├── manager.py        # Logic for JSON sync, persistence, and Pickling
│   └── fusion.py         # Logic for RRF (Reciprocal Rank Fusion)
├── goal.md              # Project roadmap and technical goals
└── SRS.txt              # This file: Architectural blueprint

File Content Blueprints
-----------------------

1. core/engines.py
   - Class: KeywordEngine
     - __init__(): Initializes BM25 with tokenized corpus.
     - search(): Returns a ranked list of document IDs and their BM25 scores.
   - Class: VectorEngine
     - __init__(): Initializes ChromaDB and Embedding model.
     - search(): Returns a ranked list of document IDs and their distances.

2. core/manager.py
   - Class: IngestionManager
     - load_json_data(): Reads all JSON files from the data/ directory.
     - sync(): Orchestrates batch ID checking in Chroma and Pickle management for BM25.
     - save_index(): Serializes the BM25 object to a Pickle file.
     - load_index(): Deserializes the BM25 object from a Pickle file.

3. core/fusion.py
   - Class: HybridRetriever
     - __init__(): Takes KeywordEngine and VectorEngine instances.
     - rrf_score(): Static method to calculate the Reciprocal Rank Fusion score.
     - search(): Executes dual-engine search and returns the fused ranked results.

4. main.py
   - Function: main()
     - Initializes the IngestionManager.
     - Sets up the HybridRetriever.
     - Runs the Streamlit UI or CLI loop for user queries.
